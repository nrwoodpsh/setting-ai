# decisions — 아키텍처 결정 기록 (ADR)

**"왜 이렇게 했는가"**를 남긴다. `task-*.md`의 History는 그 task 안의 변경만 담지만, 여기는 **프로젝트 수준의 결정**(라이브러리 선택, 패턴 채택, 트레이드오프)을 담는다.

## ADR 인덱스

이 표가 ADR의 **상시-참조 인덱스**다. **ADR을 추가하거나 상태를 바꾸면 이 표를 함께 갱신**한다.

| ADR | 제목 | 상태 | 날짜 |
|:---|:---|:---|:---|
| [0001](0001-jwt-over-session.md) | 세션 대신 JWT 사용 | 채택 | 2026-07-14 |
| [0002](0002-approval-line-rule-engine.md) | 결재선은 규칙엔진 + 예외 오버라이드 테이블 | 채택 | 2026-07-16 |

**상태값**: `제안` · `채택` · `대체됨(→NNNN)` · `폐기`.

## 원칙

- 결정은 **번호순 append**. 뒤집을 땐 새 ADR로 "0001을 대체함"이라 명시(기존 삭제 금지) + 위 인덱스에서 0001을 `대체됨(→NNNN)`으로.
- **인덱스 동기화.** 새 ADR·상태 변경은 위 인덱스 표에 즉시 반영한다 — 표와 파일이 어긋나면 그것도 드리프트다.
- 참조 허용 — 후속 작업이 결정 맥락을 알아야 할 때 `@`로 주입.
