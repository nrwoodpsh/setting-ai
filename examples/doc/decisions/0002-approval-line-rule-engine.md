# ADR 0002 — 결재선은 규칙엔진 + 예외 오버라이드 테이블로 결정

- **상태**: 채택 (2026-07-16)
- **맥락**: `approval` 도메인 신규 개발 앞서, "결재선을 자동 결정할 수 있는가"가 불확실 → `/spike`로 검증. 결론: [`analysis/spike-approval-routing-20260716.md`](../analysis/spike-approval-routing-20260716.md).
- **결정**: 결재선 자동 결정은 **직급 사다리 + 금액 임계 규칙엔진 + 예외 오버라이드 테이블(TF·특수조직·대리결재)**. 자동 96% + 사람 보정 4%를 전제로 한다.
  - 예외는 **코드가 아니라 데이터 테이블**로 관리(조직 개편 시 무배포 변경).
  - 자동 결정 결과는 **기안자 제출 전 수정 가능**(감사 요건 + 규칙 밖 4%).
- **트레이드오프**: 규칙+예외 테이블은 조직 특수 케이스가 늘면 테이블 유지비가 든다 → 오버라이드는 데이터로 두어 배포 없이 대응.
- **대안(기각)**: ① 순수 규칙엔진 — 실측 88%로 기준(95%) 미달(TF·대리결재 실패). ② ML 분류기 — 라벨 3천 건으로 학습 부족 + **감사가 요구하는 설명가능성 없음**.
- **영향**: `approval` 도메인 계약·서비스 전반. 금액 임계는 설정값. 실제 구현은 spike 코드 승격이 아니라 `/design → /builder`로 재작성.
