# decisions — 아키텍처 결정 기록 (ADR)

**"왜 이렇게 했는가"**를 남긴다. `task-*.md`의 History는 *그 task 안*의 변경만 담지만, 여기는 **프로젝트 수준의 결정**(라이브러리 선택, 패턴 채택, 트레이드오프)을 담는다. 없으면 결정 이유가 증발하고 3개월 뒤 같은 논쟁을 반복한다.

## 언제 쓰나

중요한 결정이 나올 때마다. **`/design`·`/builder`·`/sync`·`/spike`가 결정 발생 시 ADR 기록을 트리거한다**(사람 확인). 사람이 직접 추가해도 된다. 상시가 아니라 **결정이 있을 때만.**

> **신규 프로젝트의 "알파"**(목표·핵심 가설·성공 기준)도 여기에 ADR 1장으로 남긴다 — 별도 커맨드는 없다. 이게 `/spike` 검증의 출발점이 된다.

## 작성 방식

결정당 1파일: `decisions/{번호}-{제목}.md` (예: `0001-use-jwt-not-session.md`).

```markdown
# ADR 0001 — 세션 대신 JWT 사용

- **상태**: 채택 (2026-07-14)
- **맥락**: MSA 확장 예정, 서버 간 세션 공유 부담.
- **결정**: 인증은 JWT(무상태) + 리프레시 토큰.
- **트레이드오프**: 토큰 폐기 즉시성 약함 → 짧은 만료 + 블랙리스트로 보완.
- **대안(기각)**: 서버 세션 — 수평 확장 시 스티키/공유 비용.
```

## ADR 인덱스

이 표가 ADR의 **상시-참조 인덱스**다(본문은 선택 로드, 목록은 여기서 한눈에). **ADR을 추가하거나 상태를 바꾸면 이 표를 함께 갱신**한다.

| ADR | 제목 | 상태 | 날짜 |
|:---|:---|:---|:---|
| _(아직 없음)_ | | | |

**상태값**: `제안` · `채택` · `대체됨(→NNNN)` · `폐기`.

## 원칙

- 결정은 **번호순 append**. 뒤집을 땐 새 ADR로 "0001을 대체함"이라 명시(기존 삭제 금지) + 위 인덱스에서 0001을 `대체됨(→NNNN)`으로.
- **인덱스 동기화.** 새 ADR·상태 변경은 위 인덱스 표에 즉시 반영한다 — 표와 파일이 어긋나면 그것도 드리프트다.
- 참조 허용 — 후속 작업이 결정 맥락을 알아야 할 때 `@`로 주입.
