# doc/ref/patterns — 프로젝트 확정 패턴 (3층의 최하층)

이 폴더는 **AI가 참조하는 정본(SSOT)**이다. `/design`·`/builder`가 여기의 패턴을 기준으로 코드를 생성한다.

## 패턴은 3층으로 관리된다

```
① 원칙층 (중앙, 플러그인/guide)   — 스택 무관 원리. 안 변함.
② 스택 프리셋층 (중앙, presets/)  — 스택별 구체 골격. "복사해서 시작하는 씨앗".
③ 프로젝트 확정층 (여기, 로컬)     — 프리셋을 복사한 뒤 실제 코드에 맞게 조정. AI가 참조하는 정본. ← 지금 이 폴더
```

## 가장 중요한 규칙

> **AI는 언제나 이 로컬 확정본만 참조한다. 중앙 프리셋은 씨앗일 뿐 진실의 원천이 아니다.**

중앙의 "이상적 패턴"은 프로젝트 실제 코드와 반드시 벌어진다. AI가 중앙 템플릿을 참조하면 현실과 안 맞는 코드를 만든다. 그래서 여기 있는 패턴은 **이 프로젝트의 실제 코드에서 뽑아내거나, 프리셋을 복사한 뒤 실제에 맞게 다듬은 것**이어야 한다.

## 구성

| 폴더 | 내용 |
|:---|:---|
| `api-contract/` | API 구조 + 에러 계약 템플릿 (`template.ts`) |
| `task-doc/` | 설계 문서 표준 골격 (`task-template.md`) |
| `layout/` | 폴더 구조·계층 분리 규약 |
| `error-handling/` | 에러 코드 레지스트리·응답 봉투·핸들러 규약 |

## 세팅 순서 (새 프로젝트)

1. `presets/<스택>/`에서 이 폴더로 씨앗 복사.
2. 이 프로젝트의 실제 코드에 맞게 각 파일을 조정.
3. `workflow.config.json`의 `contract`를 스택에 맞게 설정.
4. 이후 `/design`·`/builder`가 이 정본을 자동 참조.
