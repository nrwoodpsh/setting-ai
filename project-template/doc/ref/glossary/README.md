# ref/glossary — 용어집 (입력 정본, 상시 참조)

도메인 용어를 정의해 **AI·사람이 같은 말을 같은 뜻으로** 쓰게 한다. AI의 용어 오용(결재/결제, member/user, 취소/반품 등)을 막는 저비용·고효과 장치.

## 작성 방식

도메인이 많으면 도메인별로 분리: `glossary/{NN}-{domain}.md` (예: `01-user.md`). 적으면 `glossary/01-terms.md` 하나로. **2자리 순번을 앞에** 붙여 숫자로 시작시키면 `README.md`가 항상 맨 아래에 온다.

```markdown
| 용어 | 영문/코드 | 정의 | 혼동 주의 |
|:---|:---|:---|:---|
| 결재 | approval | 문서·요청의 승인 절차 | ≠ 결제(payment, 돈 지불) |
| 회원 | user | 로그인 계정 보유자 | ≠ 고객(customer, 구매 이력 기준) |
```

## 원칙

- 코드 식별자(영문)와 자연어(한글)를 **함께** 적어 매핑을 고정한다.
- 새 용어가 논쟁이 되면 여기서 확정하고, 코드·문서가 이를 따른다.

## 생성·갱신 주기

백지에서 사람이 다 채우는 문서가 아니다. **검증으로 얻은 지식을 AI 초안으로 받아 다듬는다.**

- **초안**: 알파/POC(`/spike`) 후, 검증에서 나온 용어로 AI가 초안 표를 만든다(소유·확정은 사람).
- **갱신**: `/design`이 혼동 가능한 새 용어를 만나면 여기 갱신을 제안한다 — 반영 안 하면 다음 설계가 옛 용어로 샌다.
