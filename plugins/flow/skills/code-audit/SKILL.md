# 코드 감사 체크리스트 (스택 무관)

## 목적

구현·리뷰 시 스택과 무관하게 통과해야 할 **명확한 게이트 항목**을 강제한다. 스택·도메인 특화 룰은 `CLAUDE.md`·프로젝트 `patterns`에 위임.

## 적용 시점

- `/builder` 최종 감사
- `/review` 점검 도구

## 체크리스트

### 가독성·구조
- [ ] 중복 코드 — 3회 이상 반복 로직은 추출 검토
- [ ] 함수 단일 책임 — 한 함수가 두 가지 이상 하지 않는가
- [ ] 의미 있는 식별자 — `temp`·`data2`·`flag` 금지
- [ ] 데드 코드 — 이번 변경으로 쓰이지 않게 된 import·변수 정리

### 정합성
- [ ] DTO 필드 ↔ 계약 타입 일치
- [ ] 에러 코드 ↔ 계약 레지스트리 일치
- [ ] 명명 규칙 ↔ `CLAUDE.md` 일치

### 오류 처리·경계
- [ ] 외부 입력 전수 검증 (사용자·외부 API·파일)
- [ ] **발생 불가능한 시나리오에 과도한 방어 코드 금지** (토큰·복잡도 낭비)
- [ ] Null/Empty/0/음수 경계값 처리

### 테스트
- [ ] Verification 명령 Exit 0
- [ ] AC 항목별 1개 이상 단위 테스트

### 보안 (간이)
- [ ] 외부 입력의 SQL·HTML·셸 escape
- [ ] 권한 체크 (인증 + 자원 접근권)
- [ ] 민감 정보(비밀번호·토큰·개인정보) 로깅 금지

## 원칙

- **범용 코딩 룰("클린 코드로 짜라")은 넣지 않는다** — 토큰 낭비. AI는 이미 표준을 잘 따른다. 이 체크리스트는 **명확한 게이트 항목**만.
- 스택·도메인 특화 룰은 `CLAUDE.md` 또는 프로젝트 `doc/ref/patterns/`에서 관리.
- 이 스킬은 **점검 도구**다 — 발견 이슈는 리포트만, 자동 수정하지 않는다.
