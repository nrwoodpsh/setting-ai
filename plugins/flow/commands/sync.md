# /sync — 수렴 국면 (여러 문 입장, 한 문 퇴장)

코드 ↔ 문서 정합성을 회복하는 **유일한 퇴장구**다. 어떤 진입점(`/design`·`/troubleshoot`·`/analysis`·직접 수정)에서 왔든 커밋 전에 여기를 통과해 **드리프트를 막는다.** `/sync`는 문서 동기화까지만 한다 — **커밋하지 않는다** (커밋은 사용자 요청 시 `/commit`).

## 대상 확정 선언 (작업 전 필수)

```
[/sync] 변경 감지: {git diff 파일 목록, BE/FE/DB/설정 분류}
        대응 설계: doc/design/{domain}/의 task-*.md·계약
        → 이 변경을 문서에 동기화합니다.
```

## 기본 참조

- `@CLAUDE.md`
- `git diff` (스테이징+미스테이징 작업 트리 변경; 범위가 넓으면 브랜치 분기점 대비로 한정)
- 변경 파일이 속한 도메인의 `task-*.md`·계약·기존 `doc/summary/`

## 절차

1. **Diff 수집·분류**: BE/FE/DB/설정별로 변경 분류. Diff가 비어 있으면 "동기화 대상 없음" 출력 후 종료.
2. **설계 문서 갱신** (`doc/design/`) — **의도된 변경만 반영, 미설명 불일치는 정정하지 말고 리포트**:
   - **의도된 설계 변경**(구현 중 합당한 이유로 바꾼 것): 계약·`task-*.md`를 코드 기준으로 갱신하고 `History`에 **변경 사유**를 자연어로 기록. 계약은 `contract-gate` 통과.
   - **설명되지 않는 코드↔계약 불일치**(버그 의심): 계약을 덮어쓰지 말고 **불일치 리포트**만 → 사람 판단. (나쁜 구현을 SSOT로 세탁 금지)
   - `task-*.md` 구현 상태는 `[완료]`로 갱신.
   - 변경 사유 중 **프로젝트 수준 결정·정책 변경**은 `doc/decisions/`에 ADR로도 남긴다(사람 확인).
3. **요약 생성** (`doc/summary/summary-{domain}-{task}-{date}.md`):
   - 개요(Task·작업자·날짜·브랜치) / 변경사항(BE·FE·DB) / API 변경 / 특이사항(설계 대비 변경·제약·후속작업)
   - 언어는 `workflow.config.json`의 `language` 설정을 따름(기본 한국어 단일).
4. **변경 파일 리포트** 출력 후 종료. **커밋하지 않는다** — 커밋이 필요하면 사용자가 `/commit`을 호출한다.

## 가드레일

- **커밋·push 금지.** `/sync`는 문서 동기화까지만 한다. 커밋은 사용자 요청 시 `/commit`이, push·merge는 사용자가 외부 툴로.
- 코드와 계약이 불일치하면 **불일치 리포트만** 출력하고 사용자 판단 대기 (자동 정정 금지).
- 설계 문서가 부재하면 경고 후 요약본만 생성하고 종료.

> 이 수렴점과 짝을 이루는 것이 `/commit`의 `drift-check`다 — 커밋 준비 시 문서가 코드와 함께 갱신됐는지 확인해 드리프트를 잡는다.
