# /run — 파이프라인 오케스트레이션 (선택 오버레이)

당신은 파이프라인 러너다. 기존 국면(`design → builder → sync`)을 **순서대로 자동 실행**해 매번 손으로 호출하는 수고를 없앤다. 단 **사람 게이트(design 청사진 컨펌 · commit · push)와 실패 정지는 절대 건너뛰지 않는다** — 자동화하는 것은 *국면 사이의 호출*이지 *게이트*가 아니다. 이건 규율형 L4의 편의 옵션이며, 기본(국면을 하나씩 호출)을 대체하지 않는다.

## 입력

| 인자 | 동작 |
|:---|:---|
| **task (+AC)** | `design`부터 시작 — 청사진을 만들고 컨펌 대기 후 이어감 |
| **(비움)** | 직전에 **확정된 design**을 이어받아 `builder`부터 시작 |

AC가 없으면 `design`이 되묻는다(측정 가능 3개 이상 확정 후 진행).

## 대상 확정 선언 (작업 전 필수)

```
[/run] 파이프라인: design → builder → sync → ⏸(commit 직전 정지)
       대상: {task 또는 이어받을 design 경로 doc/design/{domain}/{phase}/}
       정지 지점: ① design 청사진 컨펌  ② 국면 실패 시  ③ commit 직전 (사람이 /commit)
       → 이 순서로 실행하며 위 지점에서 멈춰 사람을 기다립니다.
```

## 기본 참조

각 하위 국면이 **자기 기본 참조를 그대로 따른다**(`default-reference` 스킬) — `/run`이 따로 덮지 않는다. `@CLAUDE.md`는 공통.

## 절차

각 국면은 **해당 커맨드의 절차·가드레일을 그대로 상속**해 실행한다. 국면 사이에서만 진행/정지를 판단한다.

1. **design** *(task를 받은 경우)*: `/design`을 실행해 `task-*.md`+계약을 산출한다. 산출 후 **청사진을 사람에게 제시하고 컨펌을 기다린다** — 승인 전에는 builder로 넘어가지 않는다. (비우고 들어왔으면 직전 확정 design을 대상으로 잡고 이 단계를 건너뛴다.)
2. **builder**: `/builder`로 설계대로 구현한다. `contract-gate`·`tdd-verify`가 자동 검증. **빌드/테스트가 실패하면 즉시 멈추고 리포트한다 — 자율 수정 금지**(1회 분석 후 사람 승인 대기, §7).
3. **sync**: `/sync`로 코드↔문서 드리프트를 없애고 `summary/`를 생성한다.
4. **commit 직전 정지**: `drift-check` 결과와 변경 파일 목록을 제시하고 **멈춘다.** 커밋은 하지 않는다 — 사람이 목록을 보고 `/commit`을 호출하는 것이 게이트다.

## 종료 조건

- `sync`까지 완료 + **커밋 미리보기(drift 상태·변경 목록) 제시** 후 사람 대기, 또는
- 중간 국면 실패 시 **그 지점에서 정지 + 실패 리포트**(다음 국면으로 진행 안 함).

## 가드레일

- **자동 커밋·push·merge 금지.** commit 직전에 반드시 멈춘다 — `/run`은 `/commit`을 스스로 호출하지 않는다(§7·드리프트 §3).
- **어느 국면이든 실패 시 정지·리포트, 자율 수정 금지**(§7). 테스트를 약화·우회해 통과시키지 않는다.
- **design 청사진은 사람 컨펌 전 builder로 넘어가지 않는다.** 청사진은 사람이 승인한다.
- **각 국면의 가드레일을 상속**한다 — design=구현 코드 금지, builder=설계 임의 변경 금지, sync=드리프트 방지.
- **기본(수동 국면)을 대체하지 않는다.** 사용자가 명시 호출할 때만 도는 오버레이다. 국면을 하나씩 보고 싶으면 개별 커맨드를 쓴다.
